# Tool Backlog

> **This is a backlog, not a delivery commitment.**
>
> This list captures ~50 tool ideas gathered from the April 2026
> tool-expansion brief. It is aspirational scope. It does not commit
> Dr Mohsin to building any of these.
>
> The actual build plan lives in `docs/ROADMAP.md`. The roadmap targets
> one new tool every ~6 weeks and 10 live tools at v1.0+ by 1 April 2027.
> Items may be **moved** from this backlog into the roadmap when Dr
> Mohsin decides a topic is a higher priority than the current queue.
>
> Items on this backlog must not be shown on the public site as though
> they are available. `tools.html` surfaces them only with a
> **"Proposed"** status label and no link.

---

## Legend

- **Status** (on the backlog itself; separate from the hub-page status):
  - `idea` — listed here only; not designed
  - `scoping` — preliminary scope being drafted in a working doc
  - `ready` — scoped, sourced, waiting for a roadmap slot
- **Tags** — any that apply: Emergency, MDT, NICE, BSH, Audit, Prescribing, Interpretation
- **Primary source** — the named UK guideline or international body the tool will anchor to. If no UK source exists, this column is blank and the item needs specialist scoping.

## How to move a backlog item into the roadmap

1. Pick one item from this backlog.
2. Open a PR on `docs/ROADMAP.md` that:
   - Adds the item to the Phase-2 topic schedule table.
   - Removes it (or reorders it) from the existing six-topic list if
     replacing one.
   - Changes its status on this backlog file from `idea` to `scoping`.
3. Ship the tool over the next ~6 weeks following the tool-page template
   (`docs/templates/tool-page-template.html`).
4. When the tool ships v1.0, update:
   - this file — change status to `live` (or remove the row if you
     prefer to keep only unfinished items here).
   - `tools.html` — flip the proposed card to Live.

## What this file is not

- Not a public-facing document. It lives under `/docs/`, is not linked
  from the nav, and is not part of the site's content.
- Not a clinical-content source. Nothing on this page has been
  reviewed; it is a list of names and categories only.
- Not a commitment to NHS ratification. Only local governance
  committees ratify SOPs. These tools remain decision-support.

## Wording standard for anything that reaches the public site

When a backlog item surfaces on `tools.html` (as Proposed) or ships as
an actual page (as Live/Beta), it must carry the standard wording
defined in `docs/templates/tool-page-template.html`:

- **Proposed cards** must read:
  > *"Proposed tool. Not yet clinically validated. No clinical decision-making should be based on this placeholder."*
- **Live or Beta pages** must read:
  > *"Clinical decision-support only. Use alongside local policy, senior clinical judgement, and patient-specific context. Not for direct patient use."*

Avoid the words *approved*, *ratified*, *validated*, and *NHS-ready*
anywhere on the public site unless that status is formally true for
the specific page. Personal academic sites cannot self-award these
labels; local trust governance committees do.
